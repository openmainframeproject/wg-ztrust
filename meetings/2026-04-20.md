Rick is presenting ztrust at the VM Workshop. He will share his presentation with the group.

Rick once again addresses the request for PGP key pairs. And PKI certificates would be nice as well. The more cross signed keys we have the stronger the collection becomes.

A discussion on Certificate management, new processes set the expiration at 47 days. Do we Renew or Rekey a certificate? PKI might require the limit to start with, but most people, especially for PGP keys, might set a long expiration, if any at all. We might depend on how other programs handle the new requirements. 

We need to clarify what the goal is of ztrust. Rick defined it as 1) Provide a trust store and 2) Provide documentation. 

Lyz will look for documentation about verifying Linux code. This might be helpfull for defining ztrust procedures and documentation.

Lyz mentioned the Creative Commons License is more suited for text and/or documentation. For actual code a different license might be required.
